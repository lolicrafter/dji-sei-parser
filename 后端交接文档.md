# DJI SEI 解析后端交接文档

## 1. 项目目标

本项目用于解析 DJI 码流中的 AI SEI 数据，并通过 WebSocket 实时推送给前端叠框。

当前链路：

1. 本地 H.264 样本文件循环推流到 SRS（RTMP）。
2. 后端从 SRS 拉流，提取 Annex-B NAL，解析 SEI。
3. 后端通过 WebSocket 广播前端所需框选数据。
4. 前端订阅后实时绘制目标框。

## 3. 运行模式

### 3.1 文件模式（离线）

```bash
java -jar target/dji-sei-parser-1.0-SNAPSHOT.jar <input.h264> <output.jsonl> <parse.log>
```

### 3.2 实时 stdin 模式（仅解析）

```bash
ffmpeg ... -f h264 - | java -jar ... --stdin <output.jsonl> <parse.log>
```

### 3.3 实时 WebSocket 网关模式（当前主用）

```bash
ffmpeg ... -f h264 - \
| java -cp "target/classes:target/dependency/*" demo.DjiAiSeiH264ParserDemo \
  --stdin-ws "127.0.0.1" "18081" "output/ws-parse.log"
```

## 5. WebSocket 消息协议（前端对接）

事件名：`sei_frame`

示例：

```json
{
  "event": "sei_frame",
  "timeStampMs": 1428437,
  "trackId": 0,
  "targets": [
    {
      "id": 68,
      "type": 2,
      "state": 1,
      "cx": 5177,
      "cy": 8321,
      "w": 92,
      "h": 245,
      "cxNorm": 0.5177,
      "cyNorm": 0.8321,
      "wNorm": 0.0092,
      "hNorm": 0.0245
    }
  ]
}
```

字段说明：

- `cx/cy/w/h`：以万分比单位（相对帧宽高）。
- `cxNorm/cyNorm/wNorm/hNorm`：已经归一化到 0~1，前端可直接按容器尺寸换算。

## 7. 稳定性设计（已做）

1. 推流循环 + 异常自动重启（脚本层）。
2. 拉流解析异常自动重连（脚本层）。
3. WebSocket 推送与解析解耦（内存队列）。
4. 队列满时丢弃旧消息（防止积压）。
5. WS Server 开启端口复用 `setReuseAddr(true)`（降低重启端口占用问题）。

## 8. 常见故障排查

### 8.1 前端报 `WebSocket connection failed`

检查 WS 端口是否监听：

```bash
lsof -nP -iTCP:18081 -sTCP:LISTEN
```

预期能看到 `java ... 127.0.0.1:18081 (LISTEN)`。

### 8.2 前端视频 loading 不动

检查 SRS 和推流是否正常：

```bash
docker ps | grep srs
netstat -an | grep 1935
```

查看推流日志：

```bash
tail -n 100 output/ws-push.log
```

### 8.3 有视频无框

查看解析日志是否命中目标 payload：

```bash
tail -n 200 output/ws-parse.log
```

关注关键字：

- `payloadType=0x0007`
- `输出帧数(解析到目标payload)=...`
